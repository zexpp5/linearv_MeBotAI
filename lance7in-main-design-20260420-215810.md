## Problem Statement

为中国"信息壁垒高、流量竞争力弱"的知识付费博主（腰部，1k-100k 粉丝），提供私域内容 → AI 知识库 → 粉丝付费问答 + 自动升级到高客单服务的一站式变现平台。

目标：在不依赖流量的前提下，把博主已积累但未变现的私域一手信息（私董会、线下活动、内部直播、PDF 讲义）转化为可持续的订阅+咨询收入。

## Demand Evidence

**当前状态：B 端需求 0 证据，C 端有 1 条一手证据。**

- ✅ C 端：创始人本人就是典型用户，为某 DIY 博主（Coze / 知识星球搭建）的知识库付过费。
- ✅ 间接信号：存在至少一个腰部博主独立想到"用 AI 工具做收费知识库"并执行 —— 说明品类真实存在。
- ❌ B 端：创始人未与任何目标画像的博主沟通过"愿不愿意使用该产品"。
- ⚠️ 创始人唯一有微信的博主（学习/鸡汤类 + YouTube 成功 + Circle 社群），属于自己文档明确排除的 ❌ 类别。创始人对目标画像（创业咨询 / 外贸 / 开店实操 + 物理一手信息）的温接触 = 0。

**需求证据的行为标准（非逻辑推演）：**
- 博主主动问"什么时候能用"
- 博主愿意先付少量钱（500-1000）试用
- 博主愿意把真实粉丝（非营销号）引入产品
- 博主看到第一笔粉丝付费后主动上传更多内容

其中任何一条都能证伪当前假设，只要直接去问就有答案。

## Status Quo

**目标博主现在怎么做：**
1. VIP 群 + 小助手人工答疑（4000/月 /人）
2. 直播连麦随机接问题
3. Circle/知识星球类社群（博主→粉丝 + 粉丝→粉丝互答）
4. 小部分技术型博主自行用 Coze + 知识星球搭 DIY 知识库

**关键观察：技术型博主会 DIY（免费工具已足够），所以 MeBotAI 的真实目标是"内容有壁垒 + 时间稀缺 + 不愿折腾"的中间带博主。中间带是否存在 + 有多大 = 当前最大未知变量。**

**粉丝端痛点（创始人亲历）：** 现有社群模式效率低 —— 想解决具体问题必须翻视频、蹲直播、排队或参加线下活动。粉丝愿意为"5 秒解决第二步的问题"付费。

## Target User & Narrowest Wedge

**不是："腰部博主" —— 这是一个画像但不是一个人。**

**是：** 一个具体的博主，同时满足：
1. 有私域一手信息积累（私董会 / 线下活动 / 行业内部经验）
2. 粉丝 1k-100k
3. 流量不是他最强的武器（不是靠娱乐/情绪吸粉）
4. 时间被粉丝答疑 / 社群运营占用严重
5. 不会自己折腾 Coze 的类型
6. 愿意为"多赚 1 万/月"而不是"省 2000/月"而合作

**最窄楔子：** 1 个博主 × 1 个知识库 × 1 个付费漏斗。验证完整闭环再复制。

## Constraints

- **预算：10k 人民币**（Phase 0 主预算）。超 10k 需要主动升级决策。可能支出项：博主签约合同（若博主要求正式合同，找模板版即可，500-1000）、ASR/转录云服务（2-3k）、内容脱敏人工整理（自己做免费，外包 1-2k）、应急补足（1-2k）。
- **时间：Phase 0 窗口 5 周**（不是 4 周，不是 6 周）。5 周是对"孤狼 + 零温接触 + 无品牌"的合理估计。
- **创始人：1 人**，同时是产品经理 + 销售 + 客服 + AI 后端（Wizard of Oz 阶段）。
- **工程：暂无工程师**。Phase 0 纯人肉 + Claude projects + 微信 + 收款码。

## Premises

1. **产品不是零摩擦。** 博主必须主动上传/授权私域内容（历史 + 增量）。所有话术和 UX 围绕这个现实，不是"零摩擦"叙述。
2. **创始人对目标客户零温接入。** Phase 0 第一步是冷启动建立接触，不是打熟人电话。
3. **中间带博主（好内容 + 愿合作 + 不 DIY）的存在性和规模是未验证假设。** 这批人存量 < 50 → 商业模式不成立；存量 > 5000 → 大概率做成。这个数字当前 0 知识。
4. **PMF 证据 = 完整漏斗转化**（采纳 Codex 版）：
   - 博主愿持续供应独家内容
   - 粉丝第二周仍来提问
   - 9.9 订阅能转化到 99 会员，部分转化到 500+ 1v1

## Cross-Model Perspective

Codex（GPT-5.4, high reasoning）独立诊断的核心输出：

**Steelman**：MeBotAI 的最强形态不是"给博主做 AI 聊天框"，而是"中国私域知识付费的 AI monetization layer" —— 把知识星球、小鹅通、微信群、直播回放、PDF、课程讲义里的碎片私域内容，变成可搜索、可付费、可升级的知识产品。真正卖的是三件事：私域内容整理、粉丝问题转化、低价订阅→高价咨询的漏斗。

**关键洞察**：真正的分层变量不是粉丝量（1k-100k），而是"DIY 倾向 / 技术意愿 / 时间稀缺度"三轴。

**挑战的前提**：Codex 认为原 Premise 4（一笔交易 = PMF）过低，一次交易易被熟人关系、好奇心、创始人推动污染。真正 PMF 需要完整漏斗跑通（已采纳为修正版）。

**Claude 同意 Codex 的**：分层变量、私域内容层的定位、完整漏斗的 PMF 标准。
**Claude 部分不同意 Codex 的**：Codex 直接给出 48 小时编码原型 → Claude 认为应先用纯人肉 Wizard of Oz 验证单位经济再投入工程。

## Approaches Considered

### Approach A: 纯人肉 Wizard of Oz（贯穿 Phase 0）
- 不写代码，创始人亲自用 Claude projects 做 AI 后端
- 1 个博主 + 50/50 分成 + 微信群 + 收款码
- 验证单位经济（博主愿合作 / 粉丝愿付 / 重复 / 升级）
- 沉没成本 0，可 1 周换垂直/换博主/换话术

### Approach B: Codex 48 小时原型
- Next.js + Supabase + pgvector + Qwen/DeepSeek + 简单 RAG
- 带引用 + 置信度降级 + 1v1 升级触发
- 跳过多租户 / 自动分账 / 博主自助注册 / 花哨 agent
- 产品化可以拿给多个博主看

### Approach C: 顾问契入
- "私域知识变现顾问" + 用现成工具帮博主搭 + 3000-5000/单
- 现金流自养验证
- 品牌定位漂移风险（顾问 ↔ 平台）

## Recommended Approach

**Approach A → Approach B 串行（组合方案），5 周分 4 段。**

### Week 0（本周，Day 1-7）：冷启动 + 博主签约

目标：从冷启动到 1 个目标画像博主口头同意 50/50 分成合作。

- 列 3 个目标画像冷启动候选（不包括鸡汤朋友）
- 至少 1 条微信对话打开
- Day 7 前：1 个博主口头同意启动 → 进 Week 1
- Day 7 前：0 个博主同意 → **执行 Week 0 分支**（见下）

**Week 0 分支（冷启动失败兜底）：**
- 检查是否目标画像定义过窄 → 重新定位（保持物理一手信息核心，放宽垂直）
- 检查是否冷启动渠道选错 → 换渠道（新榜 → 小红书私信 → 付费咨询 → 老乡关系网）
- 若 Day 14 前仍 0 博主 → 停止此路径，回 Phase 1 重新评估整个假设

### Week 1-2（Day 8-21）：内容入库 + 粉丝导入

目标：博主的核心内容进 Claude projects；博主把真实粉丝（非营销号）引入产品微信群。

- **内容入库工作包（隐藏瓶颈，必须认领）：**
  - 博主提供 3-5 个代表性内容源（直播回放 + PDF 讲义 + 知识星球导出 + 私董会录音）
  - 视频/音频转文本（用 Whisper API 或现成 ASR 服务）
  - 人工脱敏（删除第三方姓名、联系方式）+ 分段标签
  - 写入 Claude projects 的 1-3 个 project（按主题分）
  - 预算工时：**15-30 小时**（这是 Wizard of Oz 最大的时间消耗项）
- **粉丝引入：**
  - 博主在自己的私域（VIP 群/朋友圈/视频号评论）发一条消息，导引 10-30 个粉丝扫码入产品群
  - 首条消息由你和博主共同起草（不能机械推销）

**Day 10 Checkpoint：** 若博主尚未开始交付内容，当面（或电话）锁定。不能让拖延成为常态。

### Week 3-4（Day 22-35）：粉丝付费测试 + 第二周返回测试

目标：让粉丝真实提问、真实付费、真实升级。

- 你亲任 AI：粉丝提问 → 你用 Claude projects 查内容 → 回答（注明"来自博主 XX 年 XX 月的直播"等引用）
- 免费 5 问后 9.9 收款码付费 → 手动开通访问
- 低置信度问题 → 触发"这个问题我要问博主本人" + 推 500+ 1v1 预约邀约
- 每天记录：提问数、付费数、低置信度触发数、博主回复率
- **Day 21 Checkpoint**：若粉丝未入群 / 未提问 / 无人付费 → 立即和博主复盘原因；不要等到 Day 35 才看数据

### Week 5（Day 36-42）：决策点 + Approach B 启动条件

- 对照 Success Criteria 评估结果
- 完整漏斗跑通 → **启动 Approach B**：48 小时实际编码 + 1-1.5 周集成/冷启动 = 约 2 周 Phase 1 首版。保留 Week 1 的博主，再加 2-3 个同画像博主。
- 漏斗在某一环断裂 → **不进 Approach B**，回 Phase 0 修正该环后再试一轮（换博主 / 换垂直 / 换话术）。代码不写。

## Open Questions

1. **中间带博主如何系统性定位和触达？** 新榜 / 小红书私信 / 行业社群 / 付费咨询切入 / 老乡朋友介绍 —— 哪条冷启动渠道 ROI 最高未知。
2. **博主内容的授权边界。** 上传后 AI 模型是否可用于训练其他博主？合同条款需要早期明确以建立信任。
3. **AI 答错责任归属（Week 1 前必决）。** 置信度降级 + 引用原文能压低但不能消除。Phase 0 选择：每条回答博主事后抽查 + 你承担答错的退款责任。不能拖到 Phase 1。
4. **Wizard of Oz 身份话术。** "我是创业者在验证产品" vs "我是你的知识库助理" —— 后者易成交但有误导风险。Phase 0 选择：对博主透明（"我在验证 AI 辅助变现的产品，你的粉丝问答由我人工运营"），对粉丝不刻意揭示（服务本质不变）。
5. **数据所有权和删除。** 博主可随时要求你删除内容 + 粉丝数据。Phase 0 实现：所有数据只在 Claude projects + 飞书文档，博主发一条微信即可触发删除。Phase 1 需要正式机制。
6. **粉丝来源（reviewer 补充）。** Phase 0 完全依赖博主从自己的私域（VIP 群、朋友圈、视频号评论）导流。关键问题：博主愿不愿意把"真实粉丝"引到一个 0 品牌的群？Week 1 首条导流消息的话术必须由博主亲自发、由你和博主共同起草。
7. **收款合规（reviewer 补充）。** 粉丝付款到创始人个人收款码 = 个人经营所得，涉及报税。Phase 0 可接受单人月流水 < 5000 元，超过需立即考虑个体工商户或公司主体。博主合作协议里明确"个人收款码仅为验证阶段使用"。
8. **响应 SLA（reviewer 补充）。** 你作为 AI 代理的在线时段 / 响应目标。Phase 0 选择：每天 9:00-23:00 响应目标 30 分钟内，超出时段明确标注"明早回复"。粉丝提问晚间爆量时的兜底 = 一条预设致歉消息 + 次日补发答案。
9. **内容入库 SLA（reviewer 补充）。** 博主交付内容后多久能让粉丝开始提问？Phase 0 目标 48-72 小时（3-5 个内容源）。时间超过意味着工作包估算错误，需重新规划。

## Success Criteria

**Phase 0（5 周）PASS = 以下 5 条全部达成（对齐修正版 Premise 4 的完整漏斗转化）：**

| # | 指标 | 目标值 | 测量方法 |
|---|------|--------|---------|
| 1 | 博主合作 | 1 个非熟人目标画像博主口头 + 微信文字确认 50/50 分成 | 微信对话截图 |
| 2 | 粉丝提问量 | ≥ 15 个真实粉丝提 ≥ 1 问题 | 产品群 + 对话记录 |
| 3 | 9.9 付费转化 | ≥ 3 个粉丝**实际付 9.9**（付费到账，非承诺） | 收款码后台 |
| 4 | 第二周返回 | ≥ 1 个付费粉丝在付费后 7-14 天内**主动**再次提问（非被动触达） | 提问时间戳对比 |
| 5 | 高客单转化 | ≥ 1 个粉丝**实际付 99 会员**，或**实际付 500+ 1v1**（非口头承诺、非邀约接受） | 收款码后台 |

**PARTIAL = 3/5 或 4/5 达成。** 进入修正试一轮而不是进 Approach B。

**FAIL = ≤ 2/5 达成。** 重新评估整个假设（目标画像、私域壁垒、漏斗设计）。

**调整的理由（对齐 reviewer 反馈）：**
- 粉丝提问数从 20 降到 15，付费数从 5 降到 3 → 孤狼 + 陌生品牌下更现实的基数
- "升级到 99/500" 严格要求**真实付费**，不是邀约接受 → 对齐 Premise 4 的"完整漏斗"
- 第二周返回必须是**主动**再次提问 → 避免用触达消息刷指标
- 指标 4 和 5 可以是同一个粉丝（留存高且升级也是一条强信号）

**失败后的诊断优先级（从大到小）：**
1. 博主不愿合作 → 冷启动渠道或目标画像错
2. 博主同意但粉丝不入群 → 博主的私域影响力高估
3. 粉丝入群但不提问 → 产品/价值主张传达错
4. 粉丝提问但不付费 → 9.9 定价/付费墙时机错
5. 粉丝付费但不返回 → 答题质量 / 内容覆盖率不够
6. 粉丝返回但不升级 → 高客单路径设计错

## The Assignment

**你本周必须完成的唯一一件事：**

今天或明天，在目标画像范围内锁定 3 个冷启动候选人（不是你的鸡汤朋友）。通过新榜 / 小红书 / 老乡介绍 / 付费咨询切入，拿到至少 1 个的微信。发第一条消息（不是 PPT，不是产品介绍），就问 3 个问题：

不是约面谈，不是发 deck，就三个问题 + 微信对话。下周办公时间把对话截图带来。

## What I noticed about how you think

- **你有强逻辑推演能力，但它是一把双刃剑。** 你对"博主零成本为什么会拒绝"的 5 段反驳是很漂亮的结构化论证 —— 但论证赢了我不等于论证赢了博主。你用分析的愉悦替代了行动的恐惧。你自己的 dbs-diagnosis 文档已经诊断过这个模式一次了，今天又在对话里复现。这是需要你自己警觉的习惯。

- **你能接受别人的数据但抗拒别人的判断。** 当我拿你自己文档（"❌ 泛泛励志/心态类"）作证据时你立即承认。当我用推理反驳你时你反而更坚持。这是创始人常见模式 —— 可以跟数据投降，跟人不能。好处是保护了你的独立判断，代价是你会过度等"硬数据"而不做"软数据"的采集（比如去跟博主聊 30 分钟）。

- **你对产品的抽象层级很高。** 从"信息定价（对标 Polymarket）"到"私有信息交易市场"再到"AI 公司/金融机构付费调用"—— 你看到的是 3-5 年后的形态。大多数创始人到不了这个高度。但当前你离第 1 个博主的第 1 笔钱还有 2 个数量级的距离。高抽象力在早期是陷阱：它让你觉得自己已经想清楚了，其实你只想清楚了远方。

- **你对第一个博主问"他是不是目标画像"，而不问"他能不能帮我学到东西"。** 这是细微但重要的区别。哪怕你鸡汤朋友不是目标客户，他的直觉反应（"这东西我想自己搭"）已经告诉你重要信息。第一阶段每一次对话都是输入，不是筛选。筛选是第二阶段的事。
